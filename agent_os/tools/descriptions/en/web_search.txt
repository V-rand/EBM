General web search. Use for open web discovery, current/news sources, or when no structured tool fits.

Before using this for academic/entity/biomedical tasks, consider:
- papers: `openalex_works(query=...)`, `arxiv_search(...)`, `pubmed_search(...)`
- known entity facts: `wikipedia_lookup(...)`
- known authoritative domain: `domain_sites(...)` then `web_search(query="keywords site:xxx.org")`
- existing session material: `workspace_search(...)`

Good web_search calls are narrow:
- exact phrase: `"42,137 households" "ordinal probit"`
- domain locked: `query="keywords site:candidate-journal.org OR site:metadata-index.org"`
- source specific: `source="news"` only for news; `source="scholar"` only when structured academic tools are insufficient.

Browser-search tactics:
- Discovery query: 2-5 high-entropy terms. Do not paste every clue into one query.
- Domain drilldown: after a promising result, search inside that domain with `site:domain.com/path` in query.
- Page drilldown: use `intitle:`, `inurl:`, `filetype:pdf`, and exact phrases to find nearby pages/PDFs on the same site.
- Exclusion: add `-reddit -quora` to query string when noisy sources dominate.

Avoid repeating broad query variants. If 2-3 web searches do not change candidates or constraints, change tool/frame/source/domain instead of keywords.

Returns `{query, results:[{title,url,content}], count, provider, next_search_hint?}` and archives to `raw_search/web_search/`.
