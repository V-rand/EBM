Externalize research strategy state. Use this to separate facts from hypotheses and to choose a path through a multi-hop question.

Common patterns:
1. `strategy_frame(nodes=[...], edges=[...], anchors=[...])`
2. `path_choice(candidate_paths=[forward, backward, anchor-first], chosen_path=..., narrowest_node=..., first_action=...)`
3. `claim_inventory(verified_facts=[...], hypotheses=[...], unknown=[...])`
4. `working_notes(...)` or `round_update(progress=..., progress_note=...)`

This tool externalizes working state that should not live only in private reasoning and returns action cards for the next move:
- problem graph: nodes, edges, anchors
- path choice: candidate paths, chosen path, narrowest node, first action, rejected paths
- claim inventory: verified facts, hypotheses, unknowns
- compact working notes: question_type, active_goal, current_action, known, unknown, failed_paths, evidence_target, exit_condition, next_move
- active constraint
- known facts already available
- reasoning paths from known facts
- candidates and their matched / failed / missing hard constraints
- evidence verdicts
- no-progress rounds and failed pivots

Typical short-answer flow:
1. start with question_model
2. for multi-hop chain tasks, strategy_frame then path_choice before committing to a route
3. claim_inventory when guesses start blending with facts
4. focus_constraint before searching a specific unresolved constraint
5. inventory_known_facts before web_search when the constraint may be solved by reasoning
6. update_candidate / add_evidence after tools or reasoning
7. round_update after each research round
8. analyze_constraint when a clue needs reasoning rather than broad search
9. working_notes every few retrieval/reasoning steps, when stuck, or before pivot/answer. Use current_action to distinguish reason, lookup, match, search, verify, and answer.
10. next_action to get control signals before searching again

If next_action returns must_inventory_known_facts, first list what is already known about the active candidate/constraint and possible reasoning paths. For discovery-first paper/document lookup, a targeted high-entropy retrieval plan may be the right next action.

If action_card blocks web_search/web_read, follow the card first. It is designed to turn "think harder" into a concrete reasoning task.
