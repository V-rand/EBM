创建子Agent执行复杂并行任务。

参数：
- task: 任务描述（需详细说明背景、目标、期望输出）
- tools: 允许子Agent使用的工具列表（可选）
- subagent_type: 子Agent类型
  - general（默认）: 所有工具可用
  - explore: 只读（仅搜索和检索工具，不能写文件）
- max_iterations: 最大推理轮次（默认 32）。校验引用等广度优先任务建议 15-20
- system_prompt: 覆盖子Agent的默认系统提示词。如 `citation_checker` 预置提示词用于引用真实性校验

返回：
- sub_task_id: 子任务 ID
- status: "completed"
- result: <task_result>...</task_result>

何时使用：
- 临床证据综合报告中多个独立维度需要并行探索（如同时检索不同数据库）
- 子问题需要多步检索、交叉验证、综合分析
- 独立子问题可以并行探索，减少主Agent上下文被冲淡
- 引用验证：spawn 子 Agent 对 draft 报告做引用真实性校验

何时不使用：
- 单次工具调用（一次搜索、读一个网页）→ 直接调用对应工具
- 自己2-3步就能完成的任务
- 子问题之间强依赖、无法独立产出结果
- 子 Agent 没有 research_state 草稿纸延续主 Agent 的思考，不适合精细迭代控制的收敛判断

规则：
- spawn 前必须拆解问题、写出清晰指令。task 描述要包含检索背景、关键词和产出格式
- 子Agent在独立 session 中运行，看不到主Agent的对话历史。所以要提供完整上下文
- 子Agent不能创建子Agent
- 多个独立子任务应并行调用（同一轮多次 spawn）
- 子 Agent 的 max_iterations 限制比主 Agent 低，不适合需要精细迭代控制和收敛判断的任务

示例：
- `spawn(task="搜索 2023 年发表的对比学习论文中关于医学图像分析的最新进展。列出 5 篇代表性论文，每篇提取方法、数据集、关键结果。", subagent_type="explore")` → 只读探索任务
- `spawn(task="统计 data/cases/ 目录下所有合同案件涉及的违约金条款类型，按合同种类分类汇总。", tools=["file_read", "file_grep"])` → 限制工具范围的子任务
