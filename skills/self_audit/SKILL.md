# Self-Audit — 输出前自审

## 这是什么

不是形式化的 QA 流程，就是写完草稿后花 20 秒快速扫一眼自己刚写的东西。大多数 EBM 报告中的错误都是"写太快了没回看"，不是"能力不够"。

## 自审清单（5 项，20 秒）

### 1. 引用真实性（最高优先级）
**每条引用逐一检查：**
- [ ] 这条引用对应的 raw_search/ 归档文件存在吗？
- [ ] PMID 是本次会话检索到的还是我从记忆里猜的？
- [ ] 有没有 "PMID: TBD" 或 "et al." 后面没有具体信息的引用？
- [ ] 引用的结论和我实际读到的内容一致吗？（不是从搜索摘要推断的）

**常见错误：**
```
❌ 引用 [1] 标记 "ESC 2023 Guidelines... PMID: TBD"
   → 没读到原文就引了。改为 "据搜索片段推断/未核实" 或删除。
❌ 引用事实来自 pubmed 搜索摘要，但写成了 "原文报道..."
   → 搜索摘要不是原文。改写为 "根据 PubMed 摘要..."
```

### 2. 数字一致性
- [ ] 同一效应量在 Clinical Bottom Line、证据链、正文中数值一致吗？
- [ ] HR 0.86 在别处有没有写成 0.68？置信区间 [0.74-0.99] 有没有在某处丢了？
- [ ] 样本量、年份、NNT 各处一致吗？

**快速检查方法：** 扫一眼报告中的每个数字，确保它只在一种语境下出现一次。如果有两个地方写了"HR 0.86"，确认它们完全一致。

### 3. 结论强度匹配证据强度
- [ ] Clinical Bottom Line 的语气是否与 GRADE 等级匹配？
  - GRADE 高 → 可以用 "证实" "有效"
  - GRADE 中 → 应该用 "可能有效" "证据支持"
  - GRADE 低 → 应该用 "有限的证据提示" "不能确定"
  - GRADE 极低 → 应该用 "证据极不充分" "无法做出判断"
- [ ] 有没有把 "关联" 说成 "因果"？观察性研究的结论不能写 "X 导致 Y"
- [ ] 有没有把 "统计学显著" 等同于 "临床重要"？

### 4. 证据链断裂点标注
- [ ] 如果某层证据缺失或薄弱，明确标注了吗？
- [ ] "证据链完整" 这个表述只在整条链无断点时使用
- [ ] 有断点时，结论是否相应降级了？

### 5. 未核实内容标注
- [ ] 任何从记忆而不是检索得出的信息，标注 "据推断" 了吗？
- [ ] 任何只看到一个来源的信息，标注 "单一来源，待交叉验证" 了吗？

## 运行方式

```
写完 draft → 加载这个 skill → 逐条过 5 项清单 → 发现问题 → 修正 → 输出
```

**不要：** 因为"看起来都挺好"就跳过某条检查。
**不要：** 发现了小问题但觉得"不严重"就不改。
**可以：** 如果 5 项全过且没有任何修正，直接输出——说明这次写得确实没问题。
