# 临床证据综合 (Clinical Evidence Synthesis)

## 这是什么

证据综合（Evidence Synthesis）是 EBM 领域的标准术语，涵盖 Cochrane 系统评价、JBI 证据总结、AHRQ 证据报告、WHO HEN 综合报告等。它不是"把找到的信息堆在一起"，而是构建一条逐级可追溯的证据链路。

## 报告结构

遵循 Cochrane / JBI / AHRQ 的 IMRaD 标准框架，临床证据综合报告包含以下章节：

### 1. 标题
PICO 导向格式：`[干预] for [人群/状况]：一项证据综合` 或直接以临床问题为标题。

### 2. 结构化摘要（≤400 字）
- **目的** — 临床问题与 PICO
- **方法** — 检索来源、纳入标准、质量评估方法
- **结果** — 关键发现、证据链简述、GRADE 总评
- **结论** — 临床意义与推荐强度

### 3. 背景与问题
- 临床背景、疾病负担、现有知识缺口
- 明确的研究问题（PICO 框架）
- 本证据综合的范围与排除

### 4. 方法
- **检索策略** — 数据库列表、检索词、时间范围、纳入/排除标准
- **质量评估** — 使用的工具（RoB 2.0 / ROBINS-I / AMSTAR 2 / AGREE II）
- **证据综合方法** — 叙述性综合 / Meta 分析 / GRADE

### 5. 结果

#### 5.1 检索结果（PRISMA 流程图）
```
数据库检索 n=← PubMed n=←, Cochrane n=←
    ↓ 去重
标题/摘要筛选 n=← → 排除 n=←
    ↓
全文获取 n=← → 排除 n=← （原因：...）
    ↓
纳入 n=←
```

#### 5.2 证据链结构（EBM 核心）
每条临床主张必须构建证据链路：

```
临床推荐 ← 指南 ← 系统评价 ← 多项 RCT ← 单项 RCT
```

#### 5.3 证据概要表（GRADE Summary of Findings）
```
| 结局指标 | 研究数 | 研究设计 | GRADE | 效应量 (95%CI) | 注释 |
|----------|--------|---------|-------|----------------|------|
| 全因死亡 | 3 RCT | RCT | ⊕⊕⊕⊕ High | HR 0.86 (0.74-0.99) | 一致结果 |
```

#### 5.4 逐项分析
每个关键结局按证据链路逐层分析。

#### 5.5 证据概览表（GRADE Evidence Profile）
对每个关键结局输出 GRADE 证据概览：

```
结局: [具体结局指标]
| 研究数 | 设计 | 偏倚风险 | 不一致性 | 间接性 | 不精确性 | 发表偏倚 | 效应量 (95%CI) | GRADE |
|--------|------|---------|---------|--------|---------|---------|----------------|-------|
| 3      | RCT  | 无严重  | 无严重   | 无严重  | 无严重   | 未检测  | RR 0.72 (0.58-0.90) | ⊕⊕⊕⊕ High |
```

#### 5.6 临床底线 (Clinical Bottom Line)
**这是整个报告最重要的 1-3 句话。** 临床医生可能只读摘要和这一段。必须直接回答原始 PICO 问题，给出可操作的结论：

```
格式: [干预] 在 [人群] 中对 [结局] 的效应是 [方向和大小]，证据链 [完整/不完整]。
若证据充分: "建议 [推荐级别]..."
若证据不足: "当前证据不足以支持或反对... 需要 [具体类型] 的进一步研究。"
```

### 6. 讨论
- 主要发现总结
- 证据链完整性与缺口
- 与现有文献的一致/冲突
- 优势与局限性
- 对临床实践的意义（如适用，按 GRADE 框架给出推荐强度：强推荐 / 弱推荐 / 不做推荐）
- 对未来研究的意义

### 7. 结论与推荐
- 按 GRADE 分级：强推荐 / 弱推荐
- 证据链状态：完整 / 部分完整 / 断裂
- 更新条件：什么新证据会改变结论

### 8. 参考文献
- GB/T 7714 格式，学术文献附加 PMID
- 每条引用必须有对应的 raw_search/ 归档

## 证据链路构建

详见系统提示词的 `<证据链路>` 部分。核心原则：

1. **自上而下检索** — Guideline → Systematic Review → RCT
2. **逐跳验证** — 指南引用了哪些系统评价？→ 系统评价纳入了哪些 RCT？→ RCT 质量如何？
3. **断链标注** — 哪一层没有证据？→ 明确标注链路在此断开并降级结论
4. **逐跳 GRADE** — 不能只给一个总 GRADE。链路中每一跳独立评级，总置信度 ≤ 最弱一跳

**证据链路完整度格式：**
```
「证据链：ESC 2024 Ⅰ 类推荐 [1] ← 3 项系统评价 [2-4] ← 6 项 RCT, n=46,969 [5-10]。
  链路完整，无断点。全链路 GRADE 高质量。」
```

## 报告写作规则

1. **报告结构服务决策** — 不是研究日志，不是过程叙述。不要写"我们搜索了"、"我们发现"
2. **每个章节回答一个中心问题** — 段落开头是主张，接着是证据，最后是限制条件
3. **不要在报告里写研究过程** — 报告是给读者看的结论，不是给自己看的笔记
4. **数字带上下文** — 每个数字附带时间、地域、样本、统计基准
5. **表格服务于比较** — 只用于对比、分类、时间线或证据矩阵，不是装饰

## 引用验证协议

初稿写入 `drafts/`，完成后验证每条引用：

1. 每条引用对应 `raw_search/` 中一份真实归档文件
2. 学术文献的 PMID/DOI 必须来自实际检索结果，不能凭记忆填写
3. 未有归档文件的引用：改为"据推断/未核实"或删除
4. 验证通过后，从 `drafts/` 移到 `research/`

## AI 写作痕迹消除

报告完成后逐段检查：

### 必须删除的空洞套话
| 模式 | 替换 |
|------|------|
| "值得注意的是"、"众所周知" | 直接进入事实 |
| "综上所述"、"总而言之" | 用具体结论替代 |
| "具有重大意义"、"里程碑式" | 精确描述：解决了什么、数据是什么 |

### 必须替换的 AI 高频词
| AI 高频词 | 改为 |
|-----------|------|
| 赋能 | 为...提供技术支持 / 帮助...实现... |
| 抓手 / 闭环 / 倒逼 / 加持 | 具体说明做了什么 |
| 持续优化 / 多维度 / 全方位 | 列出具体维度或措施 |
| 底层逻辑 | 根本原因 / 基本机制 |
