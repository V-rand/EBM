# Critically Appraised Topic (CAT) — 临床证据快评

## 写作标准（参照 AFP "How to Write an Evidence-Based Clinical Review"）

CAT 是写给临床医生看的。参照 Am Fam Physician 的临床综述写作标准，你的报告应该是：**推荐开头 → 证据解释 → 局限诚实**。

## 输出结构（这样写，不要那样写）

### 打开方式——不要用 PICO 表格，直接写推荐

**第一句话就是临床推荐。** 像 AFP 的格式：先给出推荐，然后在同一段里简要解释为什么——用自然语言，不是表。

```
写的应该是这样：

「对于初治 EGFR 突变（exon 19del 或 L858R）晚期 NSCLC 患者，osimertinib 
80mg qd 是标准一线治疗。该推荐基于 FLAURA 试验——一项多中心双盲 RCT，纳入了 
556 名患者——显示 osimertinib 相比一代 TKI 将中位 PFS 从 10.2 个月延长至 
18.9 个月（HR 0.46, 95%CI 0.37-0.57）。NCCN 指南将其列为首选方案（Category 1）。
证据等级：A（高质量 RCT + 一致 NMA 结论）。」

而不是：

## 1. 临床问题
| P | I | C | O |
|---|---|---|---|

## 2. 检索策略
数据库：PubMed, Cochrane...
```

### 正文——只展开 2-4 条关键发现，其他一律删掉

临床医生只需要知道：
1. 这个干预有多大效果？（效应量 + NNT/绝对风险）
2. 这个证据可以信吗？（证据等级 + 最关键的方法学点）
3. 对什么患者适用？（人群 + 亚组差异）
4. 有什么坑？（安全性 + 局限）

**每条发现用 1 段写完，证据等级嵌在句子里。** 不要单独做一个 GRADE 表。

```
写的应该是这样（在 CBL 段落之后）：

FLAURA 是 osimertinib 一线治疗的关键证据。这项多中心双盲试验将初治 EGFR 
突变 NSCLC 随机分配至 osimertinib 或标准 TKI（吉非替尼/厄洛替尼，n=556）。
Osimertinib 组的中位 PFS 为 18.9 个月，标准 TKI 组为 10.2 个月（HR 0.46,
P<0.001）。HR 的 95% CI（0.37-0.57）完全低于 1，且效应量大——意味着 
osimertinib 降低了 54% 的疾病进展或死亡风险。证据等级：A。

OS 获益较小但在统计学上显著（中位 38.6 vs 31.8 个月，HR 0.80, 95%CI 
0.64-1.00, P=0.046）。这里有两个需要注意的细节。第一，FLAURA 的 OS 预设
检验界值是 95.05% CI（因中期分析消耗 α），但这一差异不影响结论。第二，
对照组中有 40% 的患者在进展后交叉至 osimertinib，这意味着 OS 的真实获益
可能被低估了。由于 OS 的 CI 触及 1.0 且 P 值边界显著，我们将全因死亡的
GRADE 评为 Moderate（降 1 级为不精确性）。

而不是：

## 5. GRADE 证据评估
| 结局 | 起始 | 降级 | 最终 |
|------|------|------|------|
| PFS | High | 无 | High |
| OS | High | -1 不精确性 | Moderate |
```

注意好的写法里：GRADE 变成了自然语言（"我们将全因死亡的 GRADE 评为 Moderate"），CI 被解读了（"CI 触及 1.0 且 P 值边界显著"），交叉被讨论了（"40% 交叉，真实获益可能被低估"）。这些都不是表能做到的。

### 什么时候用表——真的需要对比多项数据的时候

以下场景用表：
- 对比 ≥3 种药物的头对头数据
- 安全性 profile（多种 AE 的发生率）
- 多个亚组效应方向不一致

以下场景不要用表：
- 只有 1-2 项研究 → 段落足够了
- 报告单个 RCT 的结果 → 读者不需要表来看懂
- 证据链 → 用箭头简写就够了
- GRADE → 写进句子里，不要单独做表

### 不需要写的——删掉

永远不要出现在 CAT 报告里的内容：
- 检索策略（数据库名、检索词、检索日期）—— 没人在乎
- PICO 作为独立表格——融入到第一段里
- 研究过程描述（"我们检索了..."）—— 读者要的是结论，不是你的工作日志
- "值得注意的是"、"综上所述"、"深入探讨"—— 全部删
- 模板空槽——没搜到指南就不要写"指南层：无"，直接跳过

### 长度——按问题复杂度调整

不要每个问题都写 200 行。如果只有 1 篇 landmark RCT 回答这个问题，3 段就够了。深度的标志不是更多行，而是对现有证据的更好解释。
