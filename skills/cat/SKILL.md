# Critically Appraised Topic (CAT) — 临床证据快评

## 核心原则

CAT 是写给**临床医生看**的，不是写给数据库看的。不是表堆，也不是 bullet points——是用**自然语言把证据讲清楚**。

**结论必须在最前面。** 临床医生可能只读前两段就做决策了。但后面必须把"为什么"说透——效应量是什么意思、GRADE 为什么是这个等级、证据链在哪断了。

适用：查房问题、治疗是否有证据、门诊快速决策。
不适用：全面系统评价（用 evidence-synthesis）、PRISMA（用 systematic-review-methodology）。

## 写作流程（多轮，禁止一轮写完就输出）

不要一轮写完就发给用户。写和改必须分开：

1. **Draft（初稿，占一轮）** — 把完整的 CBL + 证据分析写入 drafts/。先不管语言是否完美，重点是把内容写全写对。
2. **Review（回看，占一轮）** — 读自己刚写的 drafts/。检查：CBL 是否回答了 PICO？每个数字有解释？人读得通吗？引用都有来源？
3. **Revise（修改，占一轮）** — 修 Review 发现的问题。`skill_use("self-audit")` 逐项过自审清单。通过后输出给用户。

绝对不要 Draft → Revise 合并在一轮里——"边写边改"会让你看不到全局问题。

## 输出格式

### 第一段：Clinical Bottom Line（放在最上面）

用 3-5 句自然段落直接回答 PICO 问题。说清楚以下内容：

1. 干预在目标人群中对主要结局的效果（方向和大小，带 95% CI）
2. 这个效果的临床意义（NNT 是多少、绝对风险降低多少、与现有标准治疗相比如何）
3. 证据链是否完整、总 GRADE 等级
4. 临床建议（如有指南推荐则引用，如无则标注"基于目前证据"）

**好的 CBL 示例：**

> Empagliflozin 在 T2DM 合并 ASCVD 患者中可降低 MACE 风险约 14%（HR 0.86, 95%CI 0.74-0.99），基于 EMPA-REG OUTCOME 试验（n=7,020，中位随访 3.1 年）。这意味着每治疗 39 名患者 3 年可预防 1 例 MACE。此获益在心衰住院方面更显著（HR 0.65, 95%CI 0.50-0.85）。证据链完整——ESC 2023 Ⅰ类推荐 ← 3 项系统评价 ← 6 项 RCT（总 n=46,969）——GRADE High。建议：T2DM 合并 ASCVD 患者无论血糖控制如何，应考虑加用 SGLT-2 抑制剂（empagliflozin 或 canagliflozin）。

不要写的 CBL：
- "本 CAT 检索了多项文献，需要进一步研究" —— 这不是结论，是推卸
- 罗列一堆数字不解释临床意义 —— "HR 0.46" 后面必须解释这意味着什么
- 用表格代替自然语言写 CBL

### 第二段：证据链路与质量评估（用自然段落写，不要表）

解释"我们凭什么相信这个结论"。用段落而不是表格回答：

- 指南层：哪个指南，什么推荐等级，基于什么证据
- 系统评价层：哪几篇，纳入了什么，结论一致性如何
- RCT 层：关键试验的设计质量（双盲？多中心？样本量？），效应量是否一致
- 整条链的 GRADE：为什么升/降级，最薄弱的一环是什么

**好的示例：**

> **证据链路：** ESC 2023 指南对 SGLT-2 抑制剂给予了 Ⅰ 类 A 级推荐 [1]，该推荐主要基于 EMPA-REG OUTCOME [5] 和 CANVAS [6] 两项多中心双盲 RCT 以及后续的 Zelniker 2019 [4] 和 Spiazzi 2024 [3] 系统评价。Zelniker 的荟萃分析纳入了 3 项 CVOT（EMPA-REG、CANVAS、DECLARE-TIMI 58，共 34,322 例），发现 SGLT-2 抑制剂在已确诊 ASCVD 的患者中 MACE 降低 14%（HR 0.86, 95%CI 0.80-0.93），而在无 ASCVD 的患者中未见显著获益（HR 1.00, 95%CI 0.87-1.16，交互 P=0.05）。这一效应修饰（effect modification）提示 SGLT-2 抑制剂的 MACE 获益主要限于二级预防人群。Spiazzi 2024 的更大规模荟萃分析（14 项 RCT, 97,412 例）进一步确认了 MACE 降低约 11%（HR 0.89, 95%CI 0.85-0.93），全因死亡降低约 11%（HR 0.89, 0.83-0.94），且在高 KDIGO 风险组中获益更大。整体 GRADE 为 High，降级因素：无——纳入 RCT 均为多中心双盲设计，各研究间异质性低（I²≈0%），CI 窄且不跨 1。证据链完整，最薄弱的一环是 VERTIS CV（ertugliflozin）在 MACE 上未达优效（HR 0.97, 0.85-1.11），提示同类效应可能存在药物间差异。

注意这段话的特点：
- 不是表，是阅读体验流畅的自然段落
- 每个数字后面有解释（"这意味着..."）
- 效应修饰（ASCVD vs 无 ASCVD）被展开讨论了，因为有临床意义
- GRADE 为什么是 High 被解释了（"降级因素：无" + 具体理由）
- 矛盾（VERTIS 不显著）被诚实写出来了

### 第三段：关键数据展开（选最重要的写，不是全列）

如果有多个关键结局或亚组需要分层讨论，用自然段落逐个展开。只在真正需要对比时用简表：

**用表的场景（少用）：** 多药头对头对比、亚组效应相反、安全性的多个 AE 类型
**不用表的场景：** 单个研究的结果描述、单层证据链路、单一结局

### 第四段：局限性与适用性（自然段落，不是 checklist）

不要写成 "1. xxx 2. xxx 3. xxx" 的列表。写成一段文字，说清：
- 这个证据对你要看的患者适用吗？
- 最大的方法学问题是啥？
- 什么新证据出来会改变结论？

## 写作规则

1. **人话**——像你跟同事在咖啡机旁边解释一篇论文那样写
2. **每个数字有解释**——写了 "HR 0.46" 就必须写 "这在临床上意味着什么"
3. **表是辅助，段落是主体**——如果某个信息可以用一句话说清，就不要做表
4. **诚实写矛盾和不确定性**——证据有冲突就说有冲突，别强行统一
5. **删研究过程**——不写"检索了 PubMed 找到 245 篇"，只写"基于 N 篇 RCT 和 M 篇 SR"
6. **中文为主**——药物名、试验名用英文（EMPA-REG OUTCOME），其余用中文
